Shortcuts

Assign up to two shortcuts per action. Works with keyboard or mouse inputs.

Tips: press Escape to cancel capture. Scroll to use wheel shortcuts. Mouse buttons include left, middle, right, back, and forward.

Cloud Sync

Manually save or load your settings to/from Google Drive. Sign in with your Google account once. Settings are always stored locally; cloud sync is manual.

API Keys

Store API keys for rules that fetch high-res images from external services (YouTube, Twitter, etc.). Keys are stored locally and referenced in rules via {settings.keyName}.

Built-in Rules

These are hardcoded patterns and behaviors that help the extension detect and display images across various sites. Toggle individual rules on or off to customize the extension's behavior.

Custom Rules

Define custom rules to find higher-quality images for specific elements. For example, find maxresdefault thumbnails for YouTube videos or high-res product images.

URL template with placeholders like {videoId}, {id}, {src}, {href}. To return a full URL, set this to {url} and extract a url variable.
Manifest V3 blocks running user-provided JavaScript (unsafe-eval), so rules use CSP-safe extractors. Provide either a JSON array of extract steps, or shorthand (one per line): XPath, qs("selector@attr"), or closest("closestSelector", "selector", "attr"). Each step reads from one or more sources and applies a regex. The first capture group becomes the variable value.
For rules that fetch images from external APIs (YouTube profiles, Twitter, etc). url: API endpoint with placeholders. path: JSON path to extract from response (optional). headers: Custom headers (optional). Use {settings.keyName} to reference API keys stored in the API Keys section.
Executes user-provided JavaScript in a safe user script world after a match. Provide a final image URL by calling returnURL(url). The script runs on the page and can read DOM. Context variables are available in Context includes ctx and trigger (the matched element). ctx provides selector, src, href, and extracted variables; trigger is a direct DOM reference to the element caught by your CSS selector. You can return either a URL via returnURL(url) or an element via returnElement(el). If you return an element (e.g., an <img> or <picture>), the extension will automatically select the highest quality source from srcset or fall back to src. Requires the userScripts permission; Chrome may prompt once.
